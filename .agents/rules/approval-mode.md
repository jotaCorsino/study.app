# approval mode — studyhub

## current mode
frontend-first, approval-driven

## what this means
- build the app in reviewable steps
- focus first on navigable screens, structure, and interaction flow
- use mock/in-memory data in the first implementation cycles
- do not implement backend, persistence, filesystem import, or external integrations yet
- prepare architecture so these capabilities can be connected later without rework

## expected delivery style
At the end of each implementation cycle, report only:
1. what was created
2. which files changed
3. how to validate
4. the next suggested step
